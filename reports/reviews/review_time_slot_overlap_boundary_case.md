# 代码审阅报告：`time_slots_overlap()` 边界 Case

**审阅时间**：2026-04-26  
**审阅对象**：
- `src/experiments/run_single_round_mvp.py` 第 39-42 行
- `spec/06_full_dataset_generation_spec.md` 第 4 节（排课时间生成）  
**审阅人**：Kimi Agent 2.6  
**问题等级**：中等（可通过生成规范规避，无需改代码）

---

## 一、问题描述

现有 `time_slots_overlap()` 采用**纯字符串集合交集**判断冲突：

```python
def time_slots_overlap(left: str, right: str) -> bool:
    left_slots = {slot.strip() for slot in left.split("|") if slot.strip()}
    right_slots = {slot.strip() for slot in right.split("|") if slot.strip()}
    return bool(left_slots & right_slots)
```

该实现假设 `time_slot` 中的每个片段（如 `Mon-1-2`）都是**原子化、不可再分**的排课单元。

---

## 二、周末排课：完全支持，无问题

代码对 `Sat`、`Sun` 和 `Mon` 一视同仁，均为字符串比较。因此：
- 生成器输出 `Sat-1-2`、`Sun-3-4` 等格式，现有代码可直接处理，**无需任何修改**。

---

## 三、"连续多时段课程"的冲突漏判风险

若生成器输出**跨时段的连续课程**（如周一早上四节课表示为 `Mon-1-4`），则会出现以下漏判：

| 课程 A | 课程 B | 实际是否冲突 | 代码检测结果 |
|---|---|---|---|
| `Mon-1-4` | `Mon-1-2` | ❌ 冲突 | ✅ **不冲突**（字符串不同） |
| `Mon-1-4` | `Mon-3-4` | ❌ 冲突 | ✅ **不冲突**（字符串不同） |
| `Mon-1-4` | `Mon-5-6` | ✅ 不冲突 | ✅ 不冲突 |

**前两行会漏判冲突**。代码将 `Mon-1-4` 视为独立原子，不知道它在物理上包含了 `Mon-1-2` 和 `Mon-3-4` 两个时段。

---

## 四、推荐解决方案

### 方案 A（推荐）：不改代码，改生成规范

生成器将连续多时段课程拆分为原子片段输出：
- 错误：`Mon-1-4`
- 正确：`Mon-1-2|Mon-3-4`

理由：
1. `spec/06` 已定义 `|` 分隔多时段的格式（`Mon-1-2|Wed-3-4`），完全支持。
2. 对下游代码零侵入，无需修改 `time_slots_overlap()`。
3. 仿真核心在于 auction 机制和投豆行为，时间冲突是外围约束，不应为此引入复杂的时间区间解析逻辑。

### 方案 B（备选）：修改冲突检测函数

重写 `time_slots_overlap()`，引入日期解析和数值区间比较（如将 `Mon-1-4` 解析为周一 `[1,4]` 时间段）。仅在后续确实需要支持更复杂的时间表达（如跨天课程、非整数时段）时考虑。
