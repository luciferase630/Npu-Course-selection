# 研究路径总览

这个目录是给第一次读项目的人准备的。它不追求记录每一次实验细节，而是回答三个问题：

1. 我们为什么要研究投豆选课？
2. 这个项目一路怎么建模、怎么试错、怎么修正结论？
3. 现在最值得带走的东西是什么？

当前最简答案：

- **这个项目最重要的产出不是一个万能公式，而是一个可复现实验沙盒 BidFlow。**
- **公开给学生的不是精确 utility 计算器，而是一套投豆思路：看拥挤比、按课程重要性调价、做预算截断、警惕策略公开后的二阶博弈。**

## 推荐阅读顺序

1. [研究路线图](research_journey.md)  
   从流传公式开始，解释为什么要做合成市场、BA、LLM、CASS、进阶公式和策略扩散实验。
2. [建模报告入口](modeling_report_index.md)  
   把完整建模报告、公式拟合报告、CASS 报告和二阶博弈报告串起来。
3. [走过的弯路与修正](wrong_turns_and_lessons.md)  
   记录哪些结论被后续实验推翻或收紧，避免读者误读旧报告。

## 当前结论边界

- 本项目使用合成数据，不使用真实学生数据。
- 合成数据用于验证建模思路，不代表任何真实教务系统的绝对 cutoff。
- `m/n` 是最重要的公开竞争信号，但不是最终答案。
- 课程重要性只能粗略分层，不应要求学生计算精确 utility。
- 公式一旦公开，会改变市场本身；少数人知道和所有人知道是两个问题。
