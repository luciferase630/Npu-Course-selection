# 进一步研究：真实三轮选课的动态博弈

这篇报告不是当前实验结论，而是下一阶段研究议程。它要回答的是：如果现实选课不是一次性市场，而是三轮连续机制，那么现在的拥挤比公式、CASS 和 BidFlow 沙盒还缺什么。

## 1. 当前模型解决了什么，没有解决什么

当前 BidFlow 里的 `time_points=3` 更接近“一轮开奖前的三次观察和调整”：学生在同一个最终分配之前看到排队状态、调整投豆、最后统一开奖。

这已经能研究：

- 拥挤比 `m/n` 如何预测录取边界。
- 少数人知道公式时是否占优。
- CASS 是否能在同一个背景市场里省豆并提高目标学生效用。
- LLM 是否会因为保守而少选课、少浪费、但牺牲机会。

但它还没有完整建模真实的三轮选课：

```text
第一轮：投豆抢课，失败者可能拿回没中的豆子。
第二轮：继续投豆，但第一轮录满的课已经消失，同时可能开放新的高热度课程。
第三轮：补选，不一定能退课；一旦选上不喜欢的课，可能形成锁定风险。
```

所以，真实三轮机制不是同一个市场重复三次，而是一个动态状态转移问题。

## 2. 第一轮：失败退款带来的局部优势

如果第一轮规则是“投了没中会退回豆子”，那么第一轮失败者在第二轮会有一个表面优势：他们还有预算。

但这个优势是片面的，因为第一轮结束后市场状态已经变了：

- 热门且容量有限的课可能已经被选完。
- 高质量替代课减少。
- 时间表被第一轮录取结果占住，一些第二轮选择会产生冲突。
- 第一轮没抢到课的学生虽然有豆子，但面对的是更差或更窄的课程池。

因此，第一轮失败退款不是单纯好事。它同时带来两种效应：

| 效应 | 对学生的影响 |
| --- | --- |
| 流动性优势 | 失败者第二轮还有豆子可用 |
| 机会集缩水 | 可选好课减少，预算未必能换成高价值课程 |

下一阶段需要把“退款”单独建成状态变量，而不是把每轮看成同质竞价。

## 3. 第二轮：不是第一轮的重复

第二轮和第一轮最大的不同，是课程池不是同质的。

第一轮结束后，有些课已经满了，剩下来的课可能整体吸引力下降。但现实里第二轮也可能开放新的高热度课，例如品酒、艺术体验、热门通识、特殊实践课等。

这会造成一个新的结构：

```text
第二轮 = 剩余普通课 + 新开放热点课 + 第一轮失败者预算回流
```

它不是低竞争市场，也不是简单的补漏市场。第二轮可能同时出现：

- 大量低竞争剩余课。
- 少数新开热点课。
- 一批第一轮失败但预算充足的学生。
- 一批第一轮已经录课、但还想补高价值选修的学生。

这会让第二轮的录取边界呈现“两极化”：普通剩余课边界很低，新开热点课边界可能很高。

## 4. 第三轮：补选和锁定风险

第三轮如果是补选，而且不能退课，那么它和前两轮的投豆竞争完全不同。

在前两轮，学生主要面对的是“抢不到”的风险。第三轮还会出现“抢到了反而麻烦”的风险：

- 课程老师不适合自己。
- 时间占用影响后续安排。
- 课程难度或考核方式不符合预期。
- 因为不能退课，低价值课程会锁住学分、时间和注意力。

这意味着第三轮不能只看录取率。第三轮要把“被不喜欢的课锁住”作为负面结果建模。

可以把第三轮的目标写成：

```text
不是尽量多拿课，
而是在缺课风险和锁定风险之间做权衡。
```

如果策略只追求“有课上”，它可能在第三轮选中低价值课程，反而给学生套上枷锁。

## 5. 下一阶段实验设计

### 5.1 三轮状态转移实验

新增一个真正的 multi-round session：

| 状态 | 需要记录 |
| --- | --- |
| 每轮课程池 | 哪些课开放、哪些课已经满员 |
| 每轮预算 | 第一轮失败退款、成功消耗、第二轮剩余 |
| 每轮课表 | 已录课程锁定的时间槽 |
| 每轮需求 | 不同轮次的 `m/n` 和录取边界 |
| 每轮后悔风险 | 第三轮选中低价值课的锁定成本 |

### 5.2 课程开放机制实验

至少比较三种第二轮课程池：

| 情景 | 说明 |
| --- | --- |
| leftover_only | 第二轮只有第一轮没满的课 |
| hotspot_release | 第二轮额外开放少数高热通识或体验课 |
| mixed_release | 第二轮既有剩余课，也有少数新开热点和少量核心课 |

这样才能检验“第二轮失败者有预算优势”到底是否真实有效。

### 5.3 策略实验

需要新增几类 agent：

| 策略 | 行为 |
| --- | --- |
| round1_aggressive | 第一轮重投，优先锁定高价值课 |
| round1_conservative | 第一轮少投，保留预算等第二轮 |
| refund_exploiter | 第一轮试探热点，失败后利用退款冲第二轮 |
| lock_risk_averse | 第三轮宁可少选，也避免低价值锁定 |
| dynamic_cass | CASS 加入轮次、退款、课程池缩水和锁定风险 |

关键不只是比较最终录取数，而是比较：

- 高价值课程获得率。
- 必修完成率。
- 第二轮预算转化效率。
- 第三轮低价值锁定率。
- 最终课表可行性。
- 预算浪费和机会损失。

### 5.4 公式实验

当前进阶公式主要预测单轮录取边界。三轮机制下，公式需要分轮校准：

| 轮次 | 公式可能需要的修正 |
| --- | --- |
| 第一轮 | 更重视高价值课和必修课的先手价值 |
| 第二轮 | 区分 leftover 课和新开热点课，不能统一套第一轮参数 |
| 第三轮 | 从“抢边界”转向“防锁定”，减少盲目补课 |

也就是说，未来公式可能不只是一个 `m/n` 函数，而是：

```text
边界估计 = 拥挤比信号 + 轮次状态 + 课程池类型 + 退款状态 + 锁定风险
```

## 6. 对当前结论的影响

当前 README 给学生的建议仍然成立一部分：

- 看 `m/n` 是有效的。
- 低竞争课不要乱投豆。
- 必修和毕业压力课值得加安全垫。
- 投豆公式必须截断。
- 策略公开后会改变市场。

但三轮机制会让策略更复杂：

- 第一轮不一定越省越好，因为好课可能直接消失。
- 第一轮失败退款不一定是优势，因为第二轮机会集会缩水。
- 第二轮不一定只是捡漏，因为可能有新开热点课。
- 第三轮不一定“有课就上”，因为不能退课会带来锁定风险。

因此，下一阶段不能只问“这门课要投多少豆”，还要问：

```text
这一轮该不该抢？
失败后豆子会不会回来？
下一轮还有没有替代品？
选上以后能不能退出？
这门课会不会锁住我的时间和学分？
```

## 7. 结论

真实三轮选课是一个动态博弈，不是三个同质单轮市场。

第一轮的核心是先手锁定和失败退款的权衡；第二轮的核心是预算回流、课程池缩水和新热点释放；第三轮的核心是补选机会和锁定风险。

BidFlow 下一步应把三轮机制做成显式状态转移，并把 CASS 从“单轮边界响应”升级成“跨轮动态策略”。在这之前，当前公式和 CASS 只能说明单轮或固定背景下的策略价值，不能声称已经解决真实三轮选课。
