# MVP 需求说明：单轮 All-Pay 实验平台

本文定义第一阶段最小可运行实验。目标不是一次性复刻学校完整三轮选课，而是先把单轮 all-pay 机制、大模型交互、轮内多时间点修改和结果记录跑通。

## 1. 目标

MVP 要回答：

- 能否从学生-教学班效用表生成每个学生的私有上下文。
- 能否让大模型学生在多个时间点根据当前待选人数修改投豆。
- 能否按随机顺序逐个调用学生，并实时更新待选人数。
- 能否在截止时只使用最后一次投豆进行 all-pay 开奖。
- 能否完整记录交互、投豆、录取和基础指标。

## 2. 范围

MVP 只包含：

- 单轮选课。
- All-pay 支付：截止时投出的豆子无论中选与否都消耗。
- 每个学生初始预算100豆。
- 豆子只能是非负整数。
- 多个轮内时间点，默认5个。
- 每个时间点内，学生按随机顺序依次决策。
- 学生能看到课程容量和当前待选人数。
- 学生看不到任何人的投豆。
- 大模型学生无策略提示词，只知道规则、目标、个人偏好和当前交互状态。
- `E0_llm_natural_baseline`：全部学生由普通 mock/openai 代理决策。
- `E1_one_scripted_policy_agent`：1 个脚本策略学生插入普通代理市场。
- `E2_10pct_scripted_policy_agents`：10% 脚本策略学生插入普通代理市场。
- 动态预算、状态依赖豆子影子价格、基础课表约束前置检查和行为标签记录。
- 多随机种子重复运行入口，用于汇总基础指标。
- 合成数据生成器：支持 `smoke`、`medium` 和 `custom` 三种规模；`custom` 可生成例如 `10学生 × 20教学班 × 3培养方案` 的在线 LLM 小测试数据。
- 数据集目录覆盖：实验入口支持通过 `--data-dir` 指向某个已生成数据集，避免每次覆盖默认 `data/synthetic`。
- OpenAI-compatible 在线推理接入：通过环境变量配置真实模型，用同一套交互 payload 调用在线 LLM。
- 在线 LLM 注意力窗口：全量课程目录仍在平台中，单次调用默认只展示最多40个课程班，优先包含上一状态已选课程、必修课程代码相关教学班和高 `utility` 候选课。
- 带反馈重试：大模型输出违反 JSON、预算、时间冲突、同课程代码唯一或学分约束时，平台先把具体错误反馈给模型重试一次；重试仍失败才回退为保持上一状态。

MVP 暂不包含：

- 三轮现实退豆。
- 公式信息组。
- 公式回测。
- `E3` 策略提示大模型实验。
- `E4/E5` 公式信息冲击实验。
- 复杂课表冲突优化和类别学分上限修复。
- 正式数学推导。

## 3. 核心术语

- `course_id`：教学班编号，表示具体老师、时间、容量的一节课。
- `course_code`：课程代码，同一代码下可有多个教学班。
- `student_course_utility_edges.csv`：学生-教学班效用边表，是最重要的初始主观种子。
- `system_prompt`：稳定规则提示词，只描述游戏规则和硬约束。
- `student_private_context`：由偏好表生成的学生私有上下文。
- `state_snapshot`：某一时刻平台给学生看的动态状态。
- `interaction_payload`：一次大模型调用的完整输入，由系统提示词、私有上下文和动态状态组合而成。
- `bid_event`：学生在某个时间点对某个教学班的投豆修改记录。

## 4. 成功标准

MVP 完成后，应能运行一个 smoke 小规模实验，例如：

- 6名学生。
- 8个教学班。
- 5个轮内时间点。
- 每个学生每个时间点至少被调用一次。
- 截止后输出录取结果和基础指标。

运行结果必须包含：

- 每次大模型交互输入输出。
- 每次投豆修改事件。
- 截止时最终投豆。
- 课程班录取结果。
- 学生预算消耗。
- 学生最终课程效用和净效用。
- E0/E1/E2 的脚本策略学生数量和效用差指标。
- 基础行为标签计数。
- 重复实验汇总表。

此外，应能生成并运行一个在线 LLM 小规模测试数据集，例如：

- 10名学生。
- 20个教学班。
- 3个培养方案。
- 完整学生-教学班效用边表，即 `10 × 20 = 200` 条边，全部 `eligible=true`。
- 使用 `--data-dir data/synthetic/n10_c20_p3_seed42` 让实验入口读取该数据集。

`medium` 数据集生成器已作为当前 MVP 能力的一部分实现；更大规模、更多重复实验和真实 API 成本控制策略属于后续实验工程问题。

## 5. 非目标

MVP 不试图证明某个策略最优，也不生成最终给学弟学妹的投豆公式。MVP 只负责产出可信实验数据，为后续观察、数学推导和策略回测提供基础。
